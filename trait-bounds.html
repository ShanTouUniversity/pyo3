PyO3 user guide

在 Python 中使用带有 trait 约束的 Rust 函数

PyO3 允许将某些函数和类轻松地从 Rust 转换为 Python（参见转换表）。
然而，当 Rust 代码需要以实现了特定 trait 的类型作为参数时，将其转换为 Python 并不总是直接可行。

本教程解释了如何将接受 trait 作为参数的 Rust 函数转换，以便在 Python 中使用实现了相同方法的类。

这有什么用？

优点

  • 让 Python 用户也能使用你的 Rust 代码
  • 可以借助借用检查器在 Rust 中编写复杂的算法

缺点

  • 性能不如原生 Rust（需要进行类型转换，且部分代码在 Python 中运行）
  • 需要调整你的代码才能暴露出来

示例

我们以一个基本示例开始：实现一个对给定模型进行操作的优化求解器。

假设我们有一个 solve 函数，它操作一个模型并改变其状态。
该函数的参数可以是任何实现了 Model trait 的模型：

#![allow(dead_code)]
pub trait Model {
    fn set_variables(&mut self, inputs: &Vec<f64>);
    fn compute(&mut self);
    fn get_results(&self) -> Vec<f64>;
}


pub fn solve<T: Model>(model: &mut T) {
    println!("神奇的求解器，将模型变为已求解状态");
}

假设我们有以下限制：

  • 我们不能修改这段代码，因为它已在多个 Rust 模型上运行。
  • 我们还有许多 Python 模型，但由于该求解器在 Python 中不可用，无法求解它们。

将其实现在 Python 中会很繁琐且容易出错，因为所有功能已经在 Rust 中实现了。

我们如何通过 PyO3 将此求解器暴露给 Python？

为 Python 类实现 trait 约束

如果一个 Python 类实现了与 Model trait 相同的三个方法，那么逻辑上它应该可以适配使用该求解器。
但不能直接传入 Py<PyAny>，因为它并未实现 Rust trait（即使 Python 模型具有所需的方法）。

为了实现该 trait，我们必须编写一个 Rust 调用 Python 模型的包装器。
方法签名必须与 trait 一致，同时记住我们不能为了在 Python 中可用而修改原始的 Rust trait。

我们想要暴露的 Python 模型如下，它已经包含所有必需的方法：

class Model:
    def set_variables(self, inputs):
        self.inputs = inputs
    def compute(self):
        self.results = [elt**2 - 3 for elt in self.inputs]
    def get_results(self):
        return self.results

以下包装器将从 Rust 调用 Python 模型，并使用一个结构体来保存作为 PyAny 对象的模型：

#![allow(dead_code)]
use pyo3::prelude::*;
use pyo3::types::PyList;


pub trait Model {
  fn set_variables(&mut self, inputs: &Vec<f64>);
  fn compute(&mut self);
  fn get_results(&self) -> Vec<f64>;
}


struct UserModel {
    model: Py<PyAny>,
}


impl Model for UserModel {
    fn set_variables(&mut self, var: &Vec<f64>) {
        println!("Rust 正在调用 Python 设置变量");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("set_variables", (PyList::new(py, var).unwrap(),), None)
                .unwrap();
        })
    }


    fn get_results(&self) -> Vec<f64> {
        println!("Rust 正在调用 Python 获取结果");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("get_results", (), None)
                .unwrap()
                .extract()
                .unwrap()
        })
    }


    fn compute(&mut self) {
        println!("Rust 正在调用 Python 执行计算");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("compute", (), None)
                .unwrap();
        })
    }
}

现在这部分已实现，让我们将模型包装器暴露给 Python。
添加 PyO3 注解并添加构造函数：

现在我们为 trait 实现添加 PyO3 注解：

#[pymethods]
impl Model for UserModel {
    // 之前的 trait 实现
}

然而，上述代码无法编译。
编译错误如下：error: #[pymethods] cannot be used on trait impl blocks

这很麻烦！
但我们可以通过为这些函数编写第二个包装器来直接调用它们。
该包装器还将处理 Python 和 Rust 之间的类型转换。

#![allow(dead_code)]
use pyo3::prelude::*;
use pyo3::types::PyList;

pub trait Model {
  fn set_variables(&mut self, inputs: &Vec<f64>);
  fn compute(&mut self);
  fn get_results(&self) -> Vec<f64>;
}

#[pyclass]
struct UserModel {
    model: Py<PyAny>,
}

impl Model for UserModel {
 fn set_variables(&mut self, var: &Vec<f64>) {
     println!("Rust 正在调用 Python 设置变量");
     Python::attach(|py| {
         self.model.bind(py)
             .call_method("set_variables", (PyList::new(py, var).unwrap(),), None)
             .unwrap();
     })
 }

 fn get_results(&self) -> Vec<f64> {
     println!("Rust 正在调用 Python 获取结果");
     Python::attach(|py| {
         self.model
             .bind(py)
             .call_method("get_results", (), None)
             .unwrap()
             .extract()
             .unwrap()
     })
 }

 fn compute(&mut self) {
     println!("Rust 正在调用 Python 执行计算");
     Python::attach(|py| {
         self.model
             .bind(py)
             .call_method("compute", (), None)
             .unwrap();
     })

 }
}


#[pymethods]
impl UserModel {
    pub fn set_variables(&mut self, var: Vec<f64>) {
        println!("Python 调用 Rust 设置变量");
        Model::set_variables(self, &var)
    }


    pub fn get_results(&mut self) -> Vec<f64> {
        println!("Python 调用 Rust 获取结果");
        Model::get_results(self)
    }


    pub fn compute(&mut self) {
        println!("Python 调用 Rust 执行计算");
        Model::compute(self)
    }
}

此包装器处理了 PyO3 要求与 trait 之间的类型转换。
为满足 PyO3 要求，该包装器必须：

  • 返回 PyResult 类型的对象
  • 方法签名中只能使用值，不能使用引用

现在运行 Python 文件：

class Model:
    def set_variables(self, inputs):
        self.inputs = inputs
    def compute(self):
        self.results = [elt**2 - 3 for elt in self.inputs]
    def get_results(self):
        return self.results


if __name__=="__main__":
  import trait_exposure


  myModel = Model()
  my_rust_model = trait_exposure.UserModel(myModel)
  my_rust_model.set_variables([2.0])
  print("从 Python 打印值: ", myModel.inputs)
  my_rust_model.compute()
  print("通过 Rust 从 Python 打印值: ", my_rust_model.get_results())
  print("直接从 Python 打印值: ", myModel.get_results())

输出结果为：

Print value from Python:  [2.0]
Compute from Python calling Rust

Get results from Python calling Rust
Get results from Rust calling Python

我们现在已成功将实现了 Model trait 的 Rust 模型暴露给了 Python！

接下来我们将暴露 solve 函数，但在那之前，先谈谈类型错误。

Python 中的类型错误

当你在 Python 中使用类型错误时会发生什么？如何改善错误提示信息？

Python 函数参数类型错误

假设第一种情况，在你的 Python 文件中使用 my_rust_model.set_variables(2.0) 而不是 my_rust_model.set_variables([2.0])。

Rust 签名期望一个向量，对应 Python 中的列表。
如果传入单个值而非向量，会发生什么？

在执行 Python 时，我们会得到：

File "main.py", line 15, in <module>
   my_rust_model.set_variables(2)
TypeError

这是一个类型错误，Python 明确指出了问题，因此很容易识别和解决。

Python 方法签名类型错误

现在假设我们的 Model 类中某个方法的返回类型错误，例如 get_results 方法在 Rust 中预期返回一个 Vec<f64>，即 Python 中的列表。

class Model:
    def set_variables(self, inputs):
        self.inputs = inputs
    def compute(self):
        self.results = [elt**2 -3 for elt in self.inputs]
    def get_results(self):
        return self.results[0]
        #return self.results <-- 这才是正确的输出

这个调用会导致以下 panic：

pyo3_runtime.PanicException: called `Result::unwrap()` on an `Err` value: PyErr { type: Py(0x10dcf79f0, PhantomData) }

这个错误信息对不了解 Rust 的 Python 用户，或不知道使用了 PyO3 的人来说毫无帮助。

但既然我们负责将 Rust 代码提供给 Python 使用，就可以对此做些改进。问题在于我们在任何可能的地方都调用了 unwrap，因此 PyO3 的任何 panic 都会直接传递给最终用户。

让我们修改执行类型转换的代码，以便向 Python 用户提供更有帮助的错误信息：

我们在 get_results 方法中使用了以下调用以执行类型转换：

#![allow(dead_code)]
use pyo3::prelude::*;
use pyo3::types::PyList;

pub trait Model {
  fn set_variables(&mut self, inputs: &Vec<f64>);
  fn compute(&mut self);
  fn get_results(&self) -> Vec<f64>;
}

#[pyclass]
struct UserModel {
    model: Py<PyAny>,
}

impl Model for UserModel {
    fn get_results(&self) -> Vec<f64> {
        println!("Rust calling Python to get the results");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("get_results", (), None)
                .unwrap()
                .extract()
                .unwrap()
        })
    }
    fn set_variables(&mut self, var: &Vec<f64>) {
        println!("Rust calling Python to set the variables");
        Python::attach(|py| {
            self.model.bind(py)
                .call_method("set_variables", (PyList::new(py, var).unwrap(),), None)
                .unwrap();
        })
    }

    fn compute(&mut self) {
        println!("Rust calling Python to perform the computation");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("compute", (), None)
                .unwrap();
        })
    }
}

让我们逐步拆解它，以实现更好的错误处理：

#![allow(dead_code)]
use pyo3::prelude::*;
use pyo3::types::PyList;

pub trait Model {
  fn set_variables(&mut self, inputs: &Vec<f64>);
  fn compute(&mut self);
  fn get_results(&self) -> Vec<f64>;
}

#[pyclass]
struct UserModel {
    model: Py<PyAny>,
}

impl Model for UserModel {
    fn get_results(&self) -> Vec<f64> {
        println!("Get results from Rust calling Python");
        Python::attach(|py| {
            let py_result: Bound<'_, PyAny> = self
                .model
                .bind(py)
                .call_method("get_results", (), None)
                .unwrap();

            if py_result.get_type().name().unwrap() != "list" {
                panic!(
                    "Expected a list for the get_results() method signature, got {}",
                    py_result.get_type().name().unwrap()
                );
            }
            py_result.extract()
        })
        .unwrap()
    }
    fn set_variables(&mut self, var: &Vec<f64>) {
        println!("Rust calling Python to set the variables");
        Python::attach(|py| {
            let py_model = self.model.bind(py)
                .call_method("set_variables", (PyList::new(py, var).unwrap(),), None)
                .unwrap();
        })
    }

    fn compute(&mut self) {
        println!("Rust calling Python to perform the computation");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("compute", (), None)
                .unwrap();
        })
    }
}

通过这样做，你可以捕获 Python 计算的结果并检查其类型，以便在执行 unwrap 之前提供更清晰的错误信息。

当然，这并未涵盖所有可能的错误输出情况：用户可能会返回一个字符串列表，而不是浮点数列表。在这种情况下，由于 PyO3 的原因仍然会发生运行时 panic，但非 Rust 用户会更难理解错误信息。

开发者在暴露 Rust 代码时，需要自行决定在 Python 类型错误处理和改进错误消息方面投入多少精力。

最终代码

现在，让我们将 solve() 函数暴露出来，使其可以从 Python 调用。

我们不能直接将 solve 函数暴露给 Python，因为类型转换无法完成——它需要一个实现了 Model trait 的对象作为输入。

然而，UserModel 已经实现了该 trait。因此，我们可以编写一个函数包装器，接收已经被暴露给 Python 的 UserModel 作为参数，从而调用核心函数 solve。

此外，还需要将结构体声明为 pub（公共）。

#![allow(dead_code)]
fn main() {}
use pyo3::prelude::*;
use pyo3::types::PyList;

pub trait Model {
    fn set_variables(&mut self, var: &Vec<f64>);
    fn get_results(&self) -> Vec<f64>;
    fn compute(&mut self);
}

pub fn solve<T: Model>(model: &mut T) {
    println!("Magic solver that mutates the model into a resolved state");
}

#[pyfunction]
#[pyo3(name = "solve")]
pub fn solve_wrapper(model: &mut UserModel) {
    solve(model);
}

#[pyclass]
pub struct UserModel {
    model: Py<PyAny>,
}

#[pymethods]
impl UserModel {
    #[new]
    pub fn new(model: Py<PyAny>) -> Self {
        UserModel { model }
    }

    pub fn set_variables(&mut self, var: Vec<f64>) {
        println!("Set variables from Python calling Rust");
        Model::set_variables(self, &var)
    }

    pub fn get_results(&mut self) -> Vec<f64> {
        println!("Get results from Python calling Rust");
        Model::get_results(self)
    }

    pub fn compute(&mut self) {
        Model::compute(self)
    }
}

#[pymodule]
mod trait_exposure {
    #[pymodule_export]
    use super::{UserModel, solve_wrapper};
}

impl Model for UserModel {
    fn set_variables(&mut self, var: &Vec<f64>) {
        println!("Rust calling Python to set the variables");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("set_variables", (PyList::new(py, var).unwrap(),), None)
                .unwrap();
        })
    }

    fn get_results(&self) -> Vec<f64> {
        println!("Get results from Rust calling Python");
        Python::attach(|py| {
            let py_result: Bound<'_, PyAny> = self
                .model
                .bind(py)
                .call_method("get_results", (), None)
                .unwrap();

            if py_result.get_type().name().unwrap() != "list" {
                panic!(
                    "Expected a list for the get_results() method signature, got {}",
                    py_result.get_type().name().unwrap()
                );
            }
            py_result.extract()
        })
        .unwrap()
    }

    fn compute(&mut self) {
        println!("Rust calling Python to perform the computation");
        Python::attach(|py| {
            self.model
                .bind(py)
                .call_method("compute", (), None)
                .unwrap();
        })
    }
}
