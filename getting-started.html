PyO3 user guide

安装

要开始使用 PyO3，你需要三样东西：Rust 工具链、Python 环境以及构建方式。我们将在下面逐一介绍。

如果你想与 PyO3 维护者和其他 PyO3 用户交流，请考虑加入 PyO3 Discord 服务器。我们很希望能听到你初学者的体验，以便让 PyO3 对所有人都更易用！

Rust

首先，确保你的系统上已安装 Rust。如果尚未安装，请参考 此处 的说明进行安装。PyO3 支持 stable 和 nightly 版本，因此你可以选择最适合自己的版本。最低要求的 Rust 版本为 1.74。

如果你可以运行 rustc --version 且版本足够新，就可以继续了！

Python

使用 PyO3 需要至少 Python 3.7。虽然你可以直接使用系统默认的 Python 解释器，但建议使用虚拟环境。

虚拟环境

虽然你可以使用任何你喜欢的虚拟环境管理工具，但我们特别推荐使用 pyenv，尤其是当你希望为多个不同版本的 Python 进行开发或测试时。本书中的示例将使用 pyenv。pyenv 的安装说明可在此处找到：这里。（注意：要使用 pyenv activate 和 pyenv virtualenv 命令，你还需安装 pyenv-virtualenv 插件。pyenv 安装程序 可同时安装两者。）

在使用 pyenv 安装时保留源代码可能会对以后的调试有帮助，这样将来调试时就能看到原始的源文件。可以通过在 pyenv install 命令中添加 --keep 标志来实现。

例如：

pyenv install 3.12 --keep

构建

有多种构建和 Python 包管理系统可供选择，比如 setuptools-rust 或 手动构建。我们推荐使用 maturin，你可以在这里安装：maturin.rs/installation.html。maturin 是专为与 PyO3 协同开发而设计的，提供了最完整的“开箱即用”体验，特别是如果你计划发布到 PyPI。maturin 本身只是一个 Python 包，因此你可以像安装其他 Python 包一样安装它。

系统 Python：

pip install maturin --user

pipx：

pipx install maturin

pyenv：

pyenv activate pyo3
pip install maturin

poetry：

poetry add -G dev maturin

安装完成后，你可以运行 maturin --version 来确认是否正确安装。

创建新项目

首先，你应该创建一个目录和虚拟环境来存放你的新项目。这里我们将使用推荐的 pyenv：

mkdir pyo3-example
cd pyo3-example
pyenv virtualenv pyo3
pyenv local pyo3

之后，你应该安装你的构建管理工具。在此示例中，我们将使用 maturin。激活虚拟环境后，将 maturin 安装到其中：

pip install maturin

现在你可以初始化新项目了：

maturin init

如果 maturin 已经安装，你也可以直接使用它创建新项目：

maturin new -b pyo3 pyo3-example
cd pyo3-example
pyenv virtualenv pyo3
pyenv local pyo3

添加到已有项目

遗憾的是，目前 maturin 无法在已有项目中运行。因此，如果你想在已有项目中使用 Python，基本上有两个选择：

  1. 按照上述方法创建一个新项目，并将现有代码移入该项目
  2. 手动编辑项目配置文件

如果你选择第二种方式，需要注意以下几点：

Cargo.toml

请确保你希望从 Python 访问的 Rust crate 被编译为库。你也可以同时生成可执行文件，但你想从 Python 访问的代码必须位于库部分。此外，请确保 crate 类型为 cdylib，并将 PyO3 添加为依赖项，如下所示：

# 如果你的 `Cargo.toml` 中已有 [package] 信息，可以忽略这一部分！
[package]
# 这里的 `name` 是包的名称。
name = "pyo3_start"
# 这些是良好的默认设置：
version = "0.1.0"
edition = "2021"

[lib]
# 原生库的名称。这是 Python 中导入库时使用的名称（例如 `import string_sum`）。
# 如果你更改了这个名称，也必须更改 `src/lib.rs` 中 `#[pymodule]` 的名称。
name = "pyo3_example"

# "cdylib" 是生成 Python 可导入的共享库所必需的。
crate-type = ["cdylib"]

[dependencies]
pyo3 = { {{#PYO3_CRATE_VERSION}}, features = ["extension-module"] }

pyproject.toml

你也应该创建一个包含以下内容的 pyproject.toml 文件：

[build-system]
requires = ["maturin>=1,<2"]
build-backend = "maturin"

[project]
name = "pyo3_example"
requires-python = ">=3.7"
classifiers = [
    "Programming Language :: Rust",
    "Programming Language :: Python :: Implementation :: CPython",
    "Programming Language :: Python :: Implementation :: PyPy",
]

运行代码

接下来，你可以设置 Rust 代码以在 Python 中使用，如下所示；例如，可将以下代码放入 src/lib.rs：

/// 一个用 Rust 实现的 Python 模块。该函数的名称必须与 `Cargo.toml` 中的 `lib.name` 一致，
/// 否则 Python 将无法导入该模块。
#[pyo3::pymodule]
mod pyo3_example {
    use pyo3::prelude::*;

    /// 将两个数字的和格式化为字符串。
    #[pyfunction]
    fn sum_as_string(a: usize, b: usize) -> PyResult<String> {
        Ok((a + b).to_string())
    }
}

现在你可以运行 maturin develop 来准备 Python 包，之后就可以像这样使用它：

$ maturin develop
# maturin 执行编译时会输出大量进度信息...
$ python
>>> import pyo3_example
>>> pyo3_example.sum_as_string(5, 20)
'25'

有关如何在 Rust 中使用 Python 代码的更多说明，请参阅 Rust 中的 Python 页面。

Maturin 导入钩子

在开发过程中，每次代码变更后都需要运行 maturin develop 才能测试。为了简化开发流程，你可以安装 Maturin Import Hook，它会在导入含有更改的库时自动运行 maturin develop。
