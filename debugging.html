PyO3 user guide

调试

宏

PyO3 的属性（#[pyclass]、#[pymodule] 等）是过程宏，这意味着它们会重写被注解项的源代码。你可以使用以下命令查看生成的源代码，该命令还会展开其他一些内容：

cargo rustc --profile=check -- -Z unstable-options --pretty=expanded > expanded.rs; rustfmt expanded.rs

（如果尚未安装 rustfmt，你可能需要先安装它。）

你也可以通过添加 -Z trace-macros 来调试经典的 !-宏：

cargo rustc --profile=check -- -Z unstable-options --pretty=expanded -Z trace-macros > expanded.rs; rustfmt expanded.rs

请注意，这些命令需要使用 Rust 的 nightly 版本，偶尔可能会有 bug。有关更复杂且稳定的版本，请参阅 cargo expand。

使用 Valgrind 运行

Valgrind 是一种用于检测内存管理错误（如内存泄漏）的工具。

你首先需要安装 Python 的调试版本，否则 Valgrind 将无法产生可用的结果。在 Ubuntu 上，例如有 python3-dbg 包。

激活包含调试解释器的环境并重新编译。如果你使用的是 Linux，请使用 ldd 检查你的二进制文件，确认你链接的是类似 libpython3.7d.so.1.0 的库，而不是 libpython3.7.so.1.0。

下载 CPython 的抑制文件。

使用 valgrind --suppressions=valgrind-python.supp ./my-command --with-options 运行 Valgrind。

获取堆栈跟踪

调查崩溃（如段错误）的最佳起点是堆栈跟踪。你可以将 RUST_BACKTRACE=1 设置为环境变量，以在 panic! 时获取堆栈跟踪。或者，你可以使用 gdb 等调试器来排查问题。Rust 提供了一个包装器 rust-gdb，它具备检查 Rust 变量的美观打印功能。由于 PyO3 在 Python 共享对象中使用 cdylib，因此它不会在 rust-gdb 中接收到美观打印的调试钩子（rust-lang/rust#96365）。上述问题中包含启用此情况下美观打印的变通方法。

  • 如前一章节所述，链接到 Python 的调试版本
  • 运行 rust-gdb <my-binary>
  • 如果你在调查 panic!，请在 rust_panic 上设置断点（b）
  • 输入 r 来运行
  • 崩溃发生后，输入 bt 或 bt full 打印堆栈跟踪

通常运行一小段 Python 代码来测试 Rust 的某部分代码会很有帮助。

rust-gdb --args python -c "import my_package; my_package.sum_to_string(1, 2)"

在你的 Rust 代码中设置断点

开发人员调试代码的首选方法之一是设置断点。在 PyO3 中，可以通过使用 rust-gdb 或 rust-lldb 等调试器与 Python 解释器一起完成此操作。

有关如何使用 lldb 和 gdb 的更多信息，你可以阅读 lldb 文档中的 gdb 到 lldb 命令映射。

常规设置

  1. 使用调试符号编译你的扩展：

    # Debug 是 maturin 的默认选项，但你可以通过以下方式显式确保调试符号：
RUSTFLAGS="-g" maturin develop


# 对于 setuptools-rust 用户：
pip install -e .

    注意：使用调试器时，请确保 python 解析为实际的 Python 二进制文件或符号链接，而不是 shim 脚本。某些工具（如 pyenv）使用 shim 脚本，可能会干扰调试。

调试器特定设置

根据你的操作系统和偏好，你可以使用两种不同的调试器：rust-gdb 或 rust-lldb。

  1. 使用 Python 解释器启动 rust-gdb：

    rust-gdb --args python

  2. 进入 gdb 后，在你的 Rust 代码中设置断点：

    (gdb) break your_module.rs:42

  3. 运行导入并使用你的 Rust 扩展的 Python 脚本：

    # 选项 1：运行内联 Python 命令
(gdb) run -c "import your_module; your_module.your_function()"


# 选项 2：运行 Python 脚本
(gdb) run your_script.py


# 选项 3：运行 pytest 测试
(gdb) run -m pytest tests/test_something.py::TestName

  1. 使用 Python 启动 rust-lldb：

    rust-lldb -- python

  2. 在你的 Rust 代码中设置断点：

    (lldb) breakpoint set --file your_module.rs --line 42

  3. 运行你的 Python 脚本：

    # 选项 1：运行内联 Python 命令
(lldb) run -c "import your_module; your_module.your_function()"


# 选项 2：运行 Python 脚本
(lldb) run your_script.py


# 选项 3：运行 pytest 测试
(lldb) run -m pytest tests/test_something.py::TestName

使用 VS Code

带有 Rust 和 Python 扩展的 VS Code 提供了集成的调试体验：

  1. 首先，安装必要的 VS Code 扩展：

    • Rust Analyzer
    • CodeLLDB
    • Python

  2. 创建一个 .vscode/launch.json 文件，其中包含使用 LLDB Debug Launcher 的配置：

    {
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Debug PyO3",
            "type": "lldb",
            "request": "attach",
            "program": "${workspaceFolder}/.venv/bin/python",
            "pid": "${command:pickProcess}",
            "sourceLanguages": [
                "rust"
            ]
        },
        {
            "name": "Launch Python with PyO3",
            "type": "lldb",
            "request": "launch",
            "program": "${workspaceFolder}/.venv/bin/python",
            "args": ["${file}"],
            "cwd": "${workspaceFolder}",
            "sourceLanguages": ["rust"]
        },
        {
            "name": "Debug PyO3 with Args",
            "type": "lldb",
            "request": "launch",
            "program": "${workspaceFolder}/.venv/bin/python",
            "args": ["path/to/your/script.py", "arg1", "arg2"],
            "cwd": "${workspaceFolder}",
            "sourceLanguages": ["rust"]
        },
        {
            "name": "Debug PyO3 Tests",
            "type": "lldb",
            "request": "launch",
            "program": "${workspaceFolder}/.venv/bin/python",
            "args": ["-m", "pytest", "tests/your_test.py::test_function", "-v"],
            "cwd": "${workspaceFolder}",
            "sourceLanguages": ["rust"]
        }
     ]
}

    此配置支持多种调试场景：

    • 附加到正在运行的 Python 进程
    • 启动当前打开的 Python 文件
    • 使用命令行参数运行特定脚本
    • 运行 pytest 测试

  1. 通过单击行号旁的空白区域在你的 Rust 代码中设置断点。

  2. 开始调试：

    • 要附加到正在运行的 Python 进程：首先启动进程，然后选择“Debug PyO3”配置并点击“开始调试”（F5）。系统将提示你选择要附加的 Python 进程。
    • 要启动 Python 脚本：打开你的 Python 脚本，选择“Launch Python with PyO3”配置并点击“开始调试”（F5）。
    • 要运行带参数的脚本：选择“Debug PyO3 with Args”（记得用实际脚本路径和参数编辑配置）。
    • 要运行测试：选择“Debug PyO3 Tests”（根据需要编辑测试路径）。

  3. 在调试 PyO3 代码时：

    • 你可以检查 Rust 变量和数据结构
    • 使用调试控制台求值表达式
    • 使用步进控件逐行执行 Rust 代码
    • 设置条件断点用于更复杂的调试场景

高级调试配置

对于高级调试场景，你可能希望添加环境变量或启用特定的 Rust 调试标志：

{
    "name": "Debug PyO3 with Environment",
    "type": "lldb",
    "request": "launch",
    "program": "${workspaceFolder}/.venv/bin/python",
    "args": ["${file}"],
    "env": {
        "RUST_BACKTRACE": "1",
        "PYTHONPATH": "${workspaceFolder}"
    },
    "sourceLanguages": ["rust"]
}

从 Jupyter Notebook 调试

对于从 VS Code 运行的 Jupyter Notebook，你可以使用以下帮助函数来自动化启动配置：

from pathlib import Path
import os
import json
import sys




def update_launch_json(vscode_config_file_path=None):
    """更新 VSCode launch.json 以包含正确的 Jupyter 内核 PID。


    参数：
        vscode_config_file_path (str, 可选)：.vscode/launch.json 文件的路径。
            如果未提供，将使用当前工作目录。
    """
    pid = get_jupyter_kernel_pid()
    if not pid:
        print("无法确定 Jupyter 内核 PID。")
        return


    # 确定 launch.json 路径
    if vscode_config_file_path:
        launch_json_path = vscode_config_file_path
    else:
        launch_json_path = os.path.join(Path(os.getcwd()), ".vscode", "launch.json")


    # 获取 Python 解释器路径
    python_path = sys.executable


    # 默认调试器配置
    debug_config = {
        "version": "0.2.0",
        "configurations": [
            {
                "name": "Debug PyO3 (Jupyter)",
                "type": "lldb",
                "request": "attach",
                "program": python_path,
                "pid": pid,
                "sourceLanguages": ["rust"],
            },
            {
                "name": "Launch Python with PyO3",
                "type": "lldb",
                "request": "launch",
                "program": python_path,
                "args": ["${file}"],
                "cwd": "${workspaceFolder}",
                "sourceLanguages": ["rust"]
            }
        ],
    }


    # 如果 .vscode 目录不存在则创建
    try:
        os.makedirs(os.path.dirname(launch_json_path), exist_ok=True)


        # 如果 launch.json 已存在，尝试更新而不是覆盖
        if os.path.exists(launch_json_path):
            try:
                with open(launch_json_path, "r") as f:
                    existing_config = json.load(f)


                # 检查我们的配置是否已存在
                config_exists = False
                for config in existing_config.get("configurations", []):
                    if config.get("name") == "Debug PyO3 (Jupyter)":
                        config["pid"] = pid
                        config["program"] = python_path
                        config_exists = True


                if not config_exists:
                    existing_config.setdefault("configurations", []).append(debug_config["configurations"][0])


                debug_config = existing_config
            except Exception:
                # 如果读取出错，我们将直接用新配置覆盖
                pass


        with open(launch_json_path, "w") as f:
            json.dump(debug_config, f, indent=4)
        print(f"已使用 PID: {pid} 更新 launch.json，位置：{launch_json_path}")
    except Exception as e:
        print(f"更新 launch.json 时出错: {e}")




def get_jupyter_kernel_pid():
    """查找正在运行的 Jupyter 内核的进程 ID (PID)。


    返回：
        int: Jupyter 内核的进程 ID，如果未找到则返回 None。
    """
    # 检查是否在 Jupyter 环境中运行
    if 'ipykernel' in sys.modules:
        pid = os.getpid()
        print(f"Jupyter 内核 PID: {pid}")
        return pid
    else:
        print("未在 Jupyter 环境中运行。")
        return None
```要使用这些函数：

1. 在你的 Jupyter notebook 中运行包含这些函数的单元格  
2. 在一个单元格中运行 `update_launch_json()`  
3. 在 VS Code 中选择 "Debug PyO3 (Jupyter)" 配置并开始调试


## 线程安全与编译器检测工具


PyO3 试图遵循 Rust 语言级别的线程安全保证，但这并不能排除 PyO3 控制范围之外的其他代码，或由 PyO3 扩展管理的有缺陷代码引发线程安全问题。分析使用 CPython C API 的 Rust 代码是否线程安全可能非常复杂，因为许多 Python 操作都可能导致任意 Python 代码的执行。相比代码分析，使用自动化方式发现线程安全问题通常更有效。

[ThreadSanitizer](https://clang.llvm.org/docs/ThreadSanitizer.html) 是一种线程安全检查运行时工具，可用于检测由线程安全漏洞或误用线程不安全数据结构引发的数据竞争。虽然它只能检测运行时触发的数据竞争，但一旦发现问题，报告通常能准确定位问题发生的位置。

要在依赖 PyO3 的库中使用 `ThreadSanitizer`，你需要安装一个 nightly 版本的 Rust 工具链，并包含 `rust-src` 组件，因为你需要重新编译 Rust 标准库：

```bash
rustup install nightly
rustup override set nightly
rustup component add rust-src

你还需要一个用 LLVM/Clang 编译的 CPython 版本，且其所用 LLVM 主版本号必须与当前 nightly Rust 使用的相同。截至 2025 年 3 月，Rust nightly 使用的是 LLVM 20。

cpython_sanity docker 镜像 提供了一个开发环境，其中包含了预编译的 CPython 3.13 或 3.14 版本，以及可选的 NumPy 和 SciPy，全部使用 LLVM 20 和 ThreadSanitizer 编译。

激活 nightly Rust 工具链后，你可以使用以下命令通过 ThreadSanitizer 构建项目：

RUSTFLAGS="-Zsanitizer=thread" maturin develop -Zbuild-std --target x86_64-unknown-linux-gnu

如果你不在 x86_64 架构的 Linux 机器上运行，请将 x86_64-unknown-linux-gnu 替换为适用于你系统的 目标三元组。你也可以将 maturin develop 替换为 cargo test 来运行 cargo 测试。注意，cargo 使用线程池运行测试，因此 cargo 测试是发现线程安全问题的一种有效方式。

你还可以将 -Zsanitizer=thread 替换为 -Zsanitizer=address，或其他 Rust 支持的检测工具。请注意，你需要使用相应的 配置脚本选项 从源码构建 CPython，以确保 Python 与你的 Rust 代码使用相同的检测工具环境。
