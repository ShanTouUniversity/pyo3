PyO3 user guide

函数签名

#[pyfunction] 属性还接受参数来控制生成的 Python 函数如何接收参数。与 Python 一样，参数可以仅为位置参数、仅为关键字参数，或两者皆可。也可以接收 *args 参数列表和 **kwargs 字典。这些参数同样适用于 #[pymethods]，相关内容将在本指南的 Python 类 部分介绍。

默认情况下，与 Python 相同，PyO3 接受所有参数为位置或关键字参数。默认情况下所有参数都是必需的。该行为可以通过 #[pyo3(signature = (...))] 选项进行配置，该选项允许使用 Python 语法编写函数签名。

本指南的这一部分详细介绍了 #[pyo3(signature = (...))] 选项及其相关选项 #[pyo3(text_signature = "...")] 的使用方法。

使用 #[pyo3(signature = (...))]

例如，下面的函数接受任意关键字参数（Python 语法中的 **kwargs）并返回传入的参数数量：

#[pyo3::pymodule]
mod module_with_functions {
    use pyo3::prelude::*;
    use pyo3::types::PyDict;


    #[pyfunction]
    #[pyo3(signature = (**kwds))]
    fn num_kwds(kwds: Option<&Bound<'_, PyDict>>) -> usize {
        kwds.map_or(0, |dict| dict.len())
    }
}

与 Python 中一样，签名中可以包含以下结构：

  • /：位置参数分隔符，位于 / 之前的每个参数都是仅位置参数。
  • *：变长参数分隔符，位于 * 之后的每个参数都是仅关键字参数。
  • *args：“args” 是变长参数列表。args 参数的类型必须是 &Bound<'_, PyTuple>。
  • **kwargs：“kwargs” 用于接收关键字参数。kwargs 参数的类型必须是 Option<&Bound<'_, PyDict>>。
  • arg=Value：具有默认值的参数。 如果 arg 参数定义在变长参数之后，则被视为仅关键字参数。 注意 Value 必须是有效的 Rust 代码，PyO3 会直接将其插入生成的代码中而不作修改。

示例：

use pyo3::prelude::*;
use pyo3::types::{PyDict, PyTuple};

#[pyclass]
struct MyClass {
    num: i32,
}
#[pymethods]
impl MyClass {
    #[new]
    #[pyo3(signature = (num=-1))]
    fn new(num: i32) -> Self {
        MyClass { num }
    }


    #[pyo3(signature = (num=10, *py_args, name="Hello", **py_kwargs))]
    fn method(
        &mut self,
        num: i32,
        py_args: &Bound<'_, PyTuple>,
        name: &str,
        py_kwargs: Option<&Bound<'_, PyDict>>,
    ) -> String {
        let num_before = self.num;
        self.num = num;
        format!(
            "num={} (was previously={}), py_args={:?}, name={}, py_kwargs={:?} ",
            num, num_before, py_args, name, py_kwargs,
        )
    }


    fn make_change(&mut self, num: i32) -> PyResult<String> {
        self.num = num;
        Ok(format!("num={}", self.num))
    }
}

类型为 Python 的参数不能出现在签名中：

#![allow(dead_code)]
use pyo3::prelude::*;
#[pyfunction]
#[pyo3(signature = (lambda))]
pub fn simple_python_bound_function(py: Python<'_>, lambda: Py<PyAny>) -> PyResult<()> {
    Ok(())
}

注意：/ 和 * 参数（如果包含）的位置决定了处理位置和关键字参数的机制。在 Python 中：

import mymodule


mc = mymodule.MyClass()
print(mc.method(44, False, "World", 666, x=44, y=55))
print(mc.method(num=-1, name="World"))
print(mc.make_change(44))

输出结果为：

num=44 (was previously=-1), py_args=(False, 'World', 666), name=Hello, py_kwargs=Some({'x': 44, 'y': 55})
num=-1 (was previously=44), py_args=(), name=World, py_kwargs=None
num=44

注意：若要使用 struct 等关键字作为函数参数，请在签名和函数定义中同时使用“原始标识符”语法 r#struct：

#![allow(dead_code)]
use pyo3::prelude::*;
#[pyfunction(signature = (r#struct = "foo"))]
fn function_with_keyword(r#struct: &str) {
    let _ = r#struct;
    /* ... */
}

使函数签名对 Python 可见

函数签名通过 __text_signature__ 属性暴露给 Python。PyO3 会自动为每个 #[pyfunction] 和所有 #[pymethods] 从 Rust 函数生成此信息，并考虑 #[pyo3(signature = (...))] 选项所做的任何覆盖。

这种自动生成只能显示字符串、整数、布尔类型和 None 的默认参数值。任何其他默认参数将显示为 ...。（.pyi 类型存根文件通常也以相同方式使用 ... 表示默认参数。）

在需要调整自动生成的签名的情况下，可以使用 #[pyo3(text_signature)] 选项覆盖它。

下面的示例创建一个函数 add，它接受两个仅位置参数 a 和 b，其中 b 的默认值为 0。

use pyo3::prelude::*;


/// 此函数将两个无符号 64 位整数相加。
#[pyfunction]
#[pyo3(signature = (a, b=0, /))]
fn add(a: u64, b: u64) -> u64 {
    a + b
}

fn main() -> PyResult<()> {
    Python::attach(|py| {
        let fun = pyo3::wrap_pyfunction!(add, py)?;

        let doc: String = fun.getattr("__doc__")?.extract()?;
        assert_eq!(doc, "此函数将两个无符号 64 位整数相加。");

        let inspect = PyModule::import(py, "inspect")?.getattr("signature")?;
        let sig: String = inspect
            .call1((fun,))?
            .call_method0("__str__")?
            .extract()?;

        #[cfg(Py_3_8)]  // 在 3.7 版本中签名不会显示 b，上游 bug？
        assert_eq!(sig, "(a, b=0, /)");

        Ok(())
    })
}

以下 IPython 输出演示了从 Python 工具中如何看到此生成的签名：

>>> pyo3_test.add.__text_signature__
'(a, b=..., /)'
>>> pyo3_test.add?
Signature: pyo3_test.add(a, b=0, /)
Docstring: 此函数将两个无符号 64 位整数相加。
Type:      builtin_function_or_method

覆盖生成的签名

可以使用 #[pyo3(text_signature = "(<some signature>)")] 属性来覆盖默认生成的签名。

在下面的代码片段中，使用文本签名属性显式包含参数 b 的默认值 0，而不是自动生成的 ...：

use pyo3::prelude::*;


/// 此函数将两个无符号 64 位整数相加。
#[pyfunction]
#[pyo3(signature = (a, b=0, /), text_signature = "(a, b=0, /)")]
fn add(a: u64, b: u64) -> u64 {
    a + b
}

fn main() -> PyResult<()> {
    Python::attach(|py| {
        let fun = pyo3::wrap_pyfunction!(add, py)?;

        let doc: String = fun.getattr("__doc__")?.extract()?;
        assert_eq!(doc, "此函数将两个无符号 64 位整数相加。");

        let inspect = PyModule::import(py, "inspect")?.getattr("signature")?;
        let sig: String = inspect
            .call1((fun,))?
            .call_method0("__str__")?
            .extract()?;
        assert_eq!(sig, "(a, b=0, /)");

        Ok(())
    })
}

PyO3 会将注解内容原样包含为 __text_signature__。以下显示了 IPython 现在如何展示这个信息（注意 b 的默认值为 0）：

>>> pyo3_test.add.__text_signature__
'(a, b=0, /)'
>>> pyo3_test.add?
Signature: pyo3_test.add(a, b=0, /)
Docstring: 此函数将两个无符号 64 位整数相加。
Type:      builtin_function_or_method

如果完全不需要签名，使用 #[pyo3(text_signature = None)] 将禁用内置签名。下面的代码片段演示了这一点：

use pyo3::prelude::*;


/// 此函数将两个无符号 64 位整数相加。
#[pyfunction]
#[pyo3(signature = (a, b=0, /), text_signature = None)]
fn add(a: u64, b: u64) -> u64 {
    a + b
}

fn main() -> PyResult<()> {
    Python::attach(|py| {
        let fun = pyo3::wrap_pyfunction!(add, py)?;

        let doc: String = fun.getattr("__doc__")?.extract()?;
        assert_eq!(doc, "此函数将两个无符号 64 位整数相加。");
        assert!(fun.getattr("__text_signature__")?.is_none());

        Ok(())
    })
}

现在函数的 __text_signature__ 将被设置为 None，IPython 在帮助中将不再显示任何签名：

>>> pyo3_test.add.__text_signature__ == None
True
>>> pyo3_test.add?
Docstring: 此函数将两个无符号 64 位整数相加。
Type:      builtin_function_or_method

签名中的类型注解

当启用 experimental-inspect Cargo 特性时，signature 属性还可以包含类型提示：

#[cfg(feature = "experimental-inspect")] {
use pyo3::prelude::*;


#[pymodule]
pub mod example {
   use pyo3::prelude::*;


   #[pyfunction]
   #[pyo3(signature = (arg: "list[int]") -> "list[int]")]
   fn list_of_int_identity(arg: Bound<'_, PyAny>) -> Bound<'_, PyAny> {
      arg
   }
}
}

该特性启用了 PyO3 正在开发中的自动生成类型存根功能，可生成包含正确类型提示的文件：

def list_of_int_identity(arg: list[int]) -> list[int]: ...

而不是泛型形式：

import typing


def list_of_int_identity(arg: typing.Any) -> typing.Any: ...

请注意，当前类型注解必须写成 Rust 字符串。
