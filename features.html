PyO3 user guide

功能参考

PyO3 提供了许多 Cargo 功能以自定义功能。本指南的这一章节详细介绍了每一项功能。

默认情况下，仅启用 macros 功能。

扩展模块作者的功能

extension-module

在使用 PyO3 构建 Python 扩展模块时必须启用此功能。

它会告诉 PyO3 的构建脚本跳过在 Unix 平台上链接 libpython.so，因为这在某些情况下是不允许的。

更多细节请参见 构建与分发 部分。

abi3

此功能在构建 Python 扩展模块时使用，用于创建与多个 Python 版本兼容的 wheel 包。

它将 PyO3 的 API 限制为 PEP 384 所保证的、与未来 Python 版本向前兼容的 Python API 子集。

更多细节请参见 构建与分发 部分。

abi3-pyXY 系列功能

（abi3-py37、abi3-py38、abi3-py39、abi3-py310 和 abi3-py311）

这些功能是对 abi3 功能的扩展，用于指定多版本 wheel 包所支持的最低 Python 版本。

更多细节请参见 构建与分发 部分。

generate-import-lib

此实验性功能用于为 MinGW-w64 和 MSVC（交叉）编译目标生成 Python DLL 的导入库。

启用此功能后，可在无需安装目标平台的 Windows Python 发行文件的情况下，交叉编译扩展模块至任意 Windows 目标平台。

更多细节请参见 构建与分发 部分。

在 Rust 中嵌入 Python 的功能

auto-initialize

此功能会改变 Python::attach，在需要时自动初始化 Python 解释器（通过调用 Python::initialize）。

如果你未启用此功能，则应在调用其他任何 Python API 前手动调用 Python::initialize()。

高级功能

experimental-async

此功能为 #[pyfunction] 和 #[pymethods] 添加对 async fn 的支持。

该功能仍有一些未完成的优化和性能改进。为帮助完成此功能，请参阅 issue #1632 及其相关草稿 PR。

experimental-inspect

此功能为构建的二进制文件添加内省数据，这些数据可使用 pyo3-introspection crate 提取，以生成 类型存根。

此外，此功能还添加了 pyo3::inspect 模块，以及 IntoPy::type_output 和 FromPyObject::type_input API，用于为 Rust 类型生成 Python 类型“注解”。

这是为 PyO3 添加生成类型注解自动支持的第一步，但仍有工作需要完成。欢迎在 issue #2454 中提供反馈或协助。

py-clone

此功能为简化迁移而引入。发现延迟引用计数（PyO3 过去使用的方式）无法保证安全性，因此在未连接到 Python 解释器时无法 Clone Py<T> 实例（会 panic）。为避免迁移过程中无预警地引入新 panic，Clone 的实现现在由该功能控制。

pyo3_disable_reference_pool

这是一个面向性能的条件编译标志，例如通过 $RUSTFLAGS 设置，它会禁用全局引用池，减少跨 Python-Rust 边界的开销。但如果启用此功能，当未连接到 Python 解释器时 Drop 一个 Py<T> 实例会导致进程终止。

macros

此功能启用对 pyo3-macros crate 的依赖，该 crate 提供 PyO3 API 中的过程宏部分：

  • #[pymodule]
  • #[pyfunction]
  • #[pyclass]
  • #[pymethods]
  • #[derive(FromPyObject)]

同时还提供 py_run! 宏。

这些宏需要较多依赖项，而仅需使用 PyO3 进行 Python FFI 的用户可能并不需要。禁用此功能可使这些用户的构建更快，因为禁用后将不会构建这些依赖项。

此功能默认启用。要禁用，请在 Cargo.toml 中将 pyo3 条目的 default-features 设置为 false。

multiple-pymethods

此功能允许每个 #[pyclass] 拥有多个 #[pymethods] 块。

大多数用户只需为每个 #[pyclass] 使用一个 #[pymethods] 块即可。此外，该功能依赖的 inventory 在某些平台（如 Wasm）上不受支持。因此此功能默认未启用，以减少大多数用户的依赖项并加快编译速度。

更多信息请见 #[pyclass] 的实现细节。

nightly

nightly 功能需要使用 nightly 版本的 Rust 编译器。它允许 PyO3 使用 auto_traits 和 negative_impls 特性，以修复 Python::detach 函数。

resolve-config

pyo3-build-config crate 的 resolve-config 功能控制其构建脚本是否自动解析 Python 解释器 / 构建配置。此功能主要用于构建 PyO3 本身。默认情况下此功能未启用，这意味着你可以自由地将 pyo3-build-config 作为独立库使用，以读取或写入 PyO3 构建配置文件，或解析 Python 解释器的元数据。

可选依赖

这些功能启用 Python 类型与其他 Rust crate 类型之间的转换，便于无缝接入 Rust 生态系统。

anyhow

添加对 anyhow 的依赖。启用从 anyhow 的 Error 类型到 PyErr 的转换，以简化错误处理。

arc_lock

为所有扩展 lock_api::Mutex 或 parking_lot::ReentrantMutex 并被包裹在 Arc 中的互斥锁启用 PyO3 的 MutexExt trait。类似于 Arc<parking_lot::Mutex>。

bigdecimal

添加对 bigdecimal 的依赖，并启用转换至其 BigDecimal 类型。

bytes

添加对 bytes 的依赖，并启用转换至其 Bytes 类型。

chrono

添加对 chrono 的依赖。启用从 chrono 类型到 Python 的转换：

  • TimeDelta -> PyDelta
  • FixedOffset -> PyDelta
  • Utc -> PyTzInfo
  • NaiveDate -> PyDate
  • NaiveTime -> PyTime
  • DateTime -> PyDateTime

chrono-local

启用与 Local 时区的相互转换。转换时将使用 iana_time_zone::get_timezone() 确定的系统当前时区。

chrono::DateTime<Local> 可从以下任一种情况转换而来：

  • 具有 tzinfo 等同于当前系统时区的 datetime 对象。
  • “朴素”的 datetime 对象（无 tzinfo），因为惯例上朴素 datetime 对象应被视为使用系统时区。

转换为 Python 时，Local 时区信息将转换为与当前系统时区匹配的 zoneinfo.ZoneInfo。

chrono-tz

添加对 chrono-tz 的依赖。 启用与 Tz 的相互转换。

either

添加对 either 的依赖。启用转换至 either 的 Either 类型。

eyre

添加对 eyre 的依赖。启用从 eyre 的 Report 类型到 PyErr 的转换，以简化错误处理。

hashbrown

添加对 hashbrown 的依赖，并启用转换至其 HashMap 和 HashSet 类型。

indexmap添加对 indexmap 的依赖，并启用转换为它的 IndexMap 类型的功能。

jiff-02

添加对 jiff@0.2 的依赖，并要求 MSRV 为 1.70。启用从 jiff 类型到 Python 的转换：

  • SignedDuration -> PyDelta
  • TimeZone -> PyTzInfo
  • Offset -> PyTzInfo
  • Date -> PyDate
  • Time -> PyTime
  • DateTime -> PyDateTime
  • Zoned -> PyDateTime
  • Timestamp -> PyDateTime
  • ISOWeekDate -> PyDate

lock_api

添加对 lock_api 的依赖，并为所有扩展自 lock_api::Mutex 和 parking_lot::ReentrantMutex 的互斥锁（如 parking_lot 或 spin）启用 Pyo3 的 MutexExt trait。

num-bigint

添加对 num-bigint 的依赖，并启用转换为它的 BigInt 和 BigUint 类型的功能。

num-complex

添加对 num-complex 的依赖，并启用转换为它的 Complex 类型的功能。

num-rational

添加对 num-rational 的依赖，并启用转换为它的 Ratio 类型的功能。

ordered-float

添加对 ordered-float 的依赖，并启用 ordered-float 类型与 Python 之间的转换：

  • NotNan -> PyFloat
  • OrderedFloat -> PyFloat

parking-lot

添加对 parking_lot 的依赖，并为 parking_lot::Once、parking_lot::Mutex 和 parking_lot::ReentrantMutex 类型启用 Pyo3 的 OnceExt 与 MutexExt trait。

rust_decimal

添加对 rust_decimal 的依赖，并启用转换为它的 Decimal 类型的功能。

time

添加对 time 的依赖，并要求 MSRV 为 1.67.1。启用 time 类型与 Python 之间的转换：

  • Date -> PyDate
  • Time -> PyTime
  • OffsetDateTime -> PyDateTime
  • PrimitiveDateTime -> PyDateTime
  • Duration -> PyDelta
  • UtcOffset -> PyTzInfo
  • UtcDateTime -> PyDateTime

serde

通过 serde 启用 Py<T> 对象的序列化与反序列化功能。这允许在包含对 #[pyclass] 实例引用的结构体上使用 #[derive(Serialize, Deserialize)：

#[cfg(feature = "serde")]
#[allow(dead_code)]
mod serde_only {
use pyo3::prelude::*;
use serde::{Deserialize, Serialize};


#[pyclass]
#[derive(Serialize, Deserialize)]
struct Permission {
    name: String,
}


#[pyclass]
#[derive(Serialize, Deserialize)]
struct User {
    username: String,
    permissions: Vec<Py<Permission>>,
}
}

smallvec

添加对 smallvec 的依赖，并启用转换为它的 SmallVec 类型的功能。

uuid

添加对 uuid 的依赖，并启用转换为它的 Uuid 类型的功能。
