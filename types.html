PyO3 user guide

Python 对象类型

PyO3 提供了两组主要的类型来与 Python 对象进行交互。本指南的这一部分详细展开介绍这些类型以及如何选择使用哪一种。

第一组类型是所有 Python 对象都被包裹在其中的智能指针。它们是 Py<T>、Bound<'py, T> 和 Borrowed<'a, 'py, T>。下面的第一个小节将详细展开介绍这三个类型及其存在的原因。

第二组类型是填充智能指针中泛型参数 T 的类型。最常见的是 PyAny，它表示任意 Python 对象（类似于 Python 的 typing.Any）。对于许多 Python 内置类型，也有具体的类型，例如 PyList、PyDict 和 PyTuple。用户定义的 #[pyclass] 类型也属于此类别。下面的第二小节将进一步说明如何使用这些类型。

PyO3 的智能指针

PyO3 的 API 提供了三种泛型智能指针：Py<T>、Bound<'py, T> 和 Borrowed<'a, 'py, T>。对于每种类型，类型参数 T 都会由一个具体的 Python 类型填充。例如，一个 Python 列表对象可以表示为 Py<PyList>、Bound<'py, PyList> 和 Borrowed<'a, 'py, PyList>。

由于这些智能指针具有不同的生命周期参数，它们的行为也各不相同。Py<T> 没有生命周期参数，Bound<'py, T> 具有 'py 生命周期作为参数，而 Borrowed<'a, 'py, T> 具有 'py 生命周期以及一个额外的生命周期 'a，用于表示它借用数据的生命周期。（你可以在下面的子小节中阅读更多关于这些生命周期的信息。）

Python 对象是引用计数的，就像 std::sync::Arc 一样。这些智能指针的主要作用之一就是将 Python 的引用计数机制带到 Rust 的 API 中。

关于何时使用这些智能指针的建议如下：

  • 尽可能使用 Bound<'py, T>，因为它提供了最高效和完整的 API。
  • Py<T> 主要用于存储在不想或无法为 Bound<'py, T> 添加生命周期参数的 Rust struct 中。
  • Borrowed<'a, 'py, T> 几乎从不使用。它偶尔出现在 Rust 与 Python 解释器之间的边界上，例如从 Python 元组中借用数据时（这是安全的，因为元组是不可变的）。

下面的章节也会更详细地解释这些智能指针。

Py<T>

Py<T> 是 PyO3 API 中的基础智能指针。类型参数 T 表示 Python 对象的类型。这种情况非常常见的是 PyAny，意味着任何 Python 对象。

由于 Py<T> 不绑定到 'py 生命周期，所以在将 Python 对象存储在不希望或无法带有生命周期参数的 Rust struct 或 enum 中时，应使用该类型。特别是 #[pyclass] 类型不允许带有生命周期，因此 Py<T> 是在其中存储 Python 对象的正确类型。

缺乏对 'py 生命周期的绑定也有一些缺点：

  • Py<T> 上的几乎所有方法都需要 Python<'py> 令牌作为第一个参数
  • 其他功能，如 Drop，需要在运行时检查是否连接到了 Python 解释器，带来较小的性能开销

由于这些缺点，Bound<'py, T> 在 PyO3 的许多 API 中更受推荐。特别是，Bound<'py, T> 更适合作为函数参数。

要将 Py<T> 转换为 Bound<'py, T>，可使用 Py::bind 和 Py::into_bound 方法。可以使用 Bound::unbind 方法将 Bound<'py, T> 重新转换回 Py<T>。

Bound<'py, T>

Bound<'py, T> 是与 Py<T> 对应的类型，同时也绑定到了 'py 生命周期。可以将其视为等同于 Rust 元组 (Python<'py>, Py<T>)。

通过绑定到 'py 生命周期，Bound<'py, T> 可以以最高的效率提供完整的 PyO3 API。这意味着，只要携带此生命周期是可接受的，通常应使用 Bound<'py, T>，否则使用 Py<T>。

Bound<'py, T> 参与 Python 的引用计数。这表示 Bound<'py, T> 拥有一个 Python 对象。仅想借用 Python 对象的 Rust 代码应使用共享引用 &Bound<'py, T>。就像 std::sync::Arc 一样，使用 .clone() 和 drop() 会廉价地增加和减少对象的引用计数（只是在这种情况下，引用计数由 Python 解释器本身实现）。

以 Bound<'py, T> 作为 PyO3 主要 API 类型为例，考虑以下 Python 代码：

def example():
    x = list()   # 创建一个 Python 列表
    x.append(1)  # 向其中添加整数 1
    y = x        # 创建对列表的第二个引用
    del x        # 删除原始引用

使用 PyO3 的 API，特别是 Bound<'py, PyList>，这段代码转换为以下 Rust 代码：

use pyo3::prelude::*;
use pyo3::types::PyList;


fn example<'py>(py: Python<'py>) -> PyResult<()> {
    let x: Bound<'py, PyList> = PyList::empty(py);
    x.append(1)?;
    let y: Bound<'py, PyList> = x.clone(); // y 是对同一列表的新引用
    drop(x); // 释放原始引用 x
    Ok(())
}
Python::attach(example).unwrap();

或者，不带类型注解：

use pyo3::prelude::*;
use pyo3::types::PyList;


fn example(py: Python<'_>) -> PyResult<()> {
    let x = PyList::empty(py);
    x.append(1)?;
    let y = x.clone();
    drop(x);
    Ok(())
}
Python::attach(example).unwrap();

函数参数的生命周期

由于 'py 生命周期经常作为 Bound<'py, T> 智能指针的一部分出现在多个函数参数中，Rust 编译器通常需要对输入和输出的生命周期进行标注。当函数输出至少有一个生命周期，而输入中存在多个生命周期时，这种情况就会发生。

举例说明，考虑这个接受 Python 对象并对其应用Python + 操作的函数：

use pyo3::prelude::*;
fn add(left: &'_ Bound<'_, PyAny>, right: &'_ Bound<'_, PyAny>) -> PyResult<Bound<'_, PyAny>> {
    left.add(right)
}

由于 Python + 操作可能会引发异常，此函数返回 PyResult<Bound<'_, PyAny>>。它不需要拥有输入的所有权，因此采用 &Bound<'_, PyAny> 共享引用。为了说明问题，所有生命周期都使用通配符 '_，让 Rust 编译器尝试推断它们。由于有四个输入生命周期（两个共享引用的生命周期，以及 Bound<'_, PyAny> 指针内部两个未命名的 'py 生命周期），编译器无法判断哪个必须与输出关联。

正确的解决方法是将 'py 生命周期作为函数的参数，并命名 Bound<'py, PyAny> 智能指针内的所有 'py 生命周期。对于共享引用，也可以将 &'_ 简化为 &。最终可行的结果如下：

use pyo3::prelude::*;
fn add<'py>(
    left: &Bound<'py, PyAny>,
    right: &Bound<'py, PyAny>,
) -> PyResult<Bound<'py, PyAny>> {
    left.add(right)
}
Python::attach(|py| {
    let s = pyo3::types::PyString::new(py, "s");
    assert!(add(&s, &s).unwrap().eq("ss").unwrap());
})

如果为 'py 生命周期命名使函数签名变得不必要的复杂，也可以接受返回无生命周期的 Py<PyAny>。代价是在实现上略微增加复杂性，如引入对 Bound::unbind 的调用：

use pyo3::prelude::*;
fn add(left: &Bound<'_, PyAny>, right: &Bound<'_, PyAny>) -> PyResult<Py<PyAny>> {
    let output: Bound<'_, PyAny> = left.add(right)?;
    Ok(output.unbind())
}
Python::attach(|py| {
    let s = pyo3::types::PyString::new(py, "s");
    assert!(add(&s, &s).unwrap().bind(py).eq("ss").unwrap());
})

Borrowed<'a, 'py, T>

Borrowed<'a, 'py, T> 是一种高级类型，仅偶尔用于与 Python 解释器交互的边界处。可以将其视为类似于共享引用 &'a Bound<'py, T>。不同之处在于，Borrowed<'a, 'py, T> 本身只是一个智能指针，而不是“指向智能指针的引用”，这在与 Python 解释器的特定交互中减少了间接层级，是有益的优化。

Borrowed<'a, 'py, T> 会解引用到 Bound<'py, T>，因此 Bound<'py, T> 上的所有方法在 Borrowed<'a, 'py, T> 上都可用。

一个使用 Borrowed<'a, 'py, T> 的例子是 PyTupleMethods::get_borrowed_item：

use pyo3::prelude::*;
use pyo3::types::PyTuple;


fn example<'py>(py: Python<'py>) -> PyResult<()> {
// 创建一个包含元素 (0, 1, 2) 的新元组
let t = PyTuple::new(py, [0, 1, 2])?;
for i in 0..=2 {
    let entry: Borrowed<'_, 'py, PyAny> = t.get_borrowed_item(i)?;
    // `PyAnyMethods::extract` 可通过解引用到 `Bound` 在 `Borrowed` 上使用
    let value: usize = entry.extract()?;
    assert_eq!(i, value);
}
Ok(())
}
Python::attach(example).unwrap();

在智能指针类型之间转换

要在 Py<T> 和 Bound<'py, T> 之间转换，请使用 bind() / into_bound() 方法。使用 as_unbound() / unbind() 方法从 Bound<'py, T> 转回 Py<T>。

let obj: Py<PyAny> = ...;
let bound: &Bound<'py, PyAny> = obj.bind(py);
let bound: Bound<'py, PyAny> = obj.into_bound(py);


let obj: &Py<PyAny> = bound.as_unbound();
let obj: Py<PyAny> = bound.unbind();
```要将 `Bound<'py, T>` 转换为 `Borrowed<'a, 'py, T>`，请使用 `as_borrowed()` 方法。`Borrowed<'a, 'py, T>` 可通过 Deref 强制解引用为 `Bound<'py, T>`。使用 `to_owned()` 方法可以增加 Python 引用计数，并从 `Borrowed<'a, 'py, T>` 创建一个新的 `Bound<'py, T>`。

```rust,ignore
let bound: Bound<'py, PyAny> = ...;
let borrowed: Borrowed<'_, 'py, PyAny> = bound.as_borrowed();


// Deref 强制转换
let bound: &Bound<'py, PyAny> = &borrowed;


// 通过增加 Python 引用计数创建新的 Bound
let bound: Bound<'py, PyAny> = borrowed.to_owned();

要将 Py<T> 与 Borrowed<'a, 'py, T> 之间转换，请使用 bind_borrowed() 方法。从 Borrowed<'a, 'py, T> 返回到 Py<T> 时，可通过 Bound<'py, T> 使用 as_unbound() 或 .to_owned().unbind() 方法。

let obj: Py<PyAny> = ...;
let borrowed: Borrowed<'_, 'py, PyAny> = obj.bind_borrowed(py);


// 通过 Deref 强制转为 Bound，再调用 Bound::as_unbound
let obj: &Py<PyAny> = borrowed.as_unbound();


// 通过增加 Python 引用计数创建新的 Bound，再解绑
let obj: Py<PyAny> = borrowed.to_owned().unbind();

具体的 Python 类型

在 Py<T>、Bound<'py, T> 和 Borrowed<'a, 'py, T> 中，类型参数 T 表示智能指针所引用的 Python 对象的类型。

类型参数 T 可以是以下类型：

  • PyAny，表示任意 Python 对象；
  • 原生 Python 类型，例如 PyList、PyTuple 和 PyDict；
  • 使用 #[pyclass] 从 Rust 定义的类型。

以下小节详细介绍如何使用这些类型：

  • 这些具体类型可使用的 API；
  • 如何将 Bound<'py, T> 转换为特定的具体类型；
  • 如何从 Bound<'py, T> 提取 Rust 数据。

使用具体 Python 类型的 API

每个具体 Python 类型（如 PyAny、PyTuple 和 PyDict）都会在对应的绑定智能指针 Bound<'py, PyAny>、Bound<'py, PyTuple> 和 Bound<'py, PyDict> 上暴露其 API。

每种类型的 API 都通过一个 trait 暴露：PyAnyMethods、PyTupleMethods、PyDictMethods 等，适用于所有具体类型。使用 trait 而非直接在 Bound 智能指针上定义关联方法，出于以下几个原因：

  • 文档清晰性：每个 trait 在 PyO3 API 文档中都有独立的页面。如果所有方法都直接放在 Bound 上，PyO3 绝大多数 API 将集中在一个极长的文档页面上。
  • 一致性：下游代码在为现有 Python 类型实现 Rust API 时，也可以遵循这种 trait 模式。此外，下游代码无法直接为 Bound 类型添加新的关联方法。
  • 未来设计：希望未来的 Rust 支持 任意 self 类型 后，可以不再需要这些 trait，而直接将方法放在 PyAny、PyTuple、PyDict 等类型上。

这些 trait 都包含在 pyo3::prelude 模块中。因此，当使用通配符导入 use pyo3::prelude::* 时，完整的 PyO3 API 就对下游代码可用。

以下函数使用 PyListMethods trait 中的 .get_item() 方法访问输入 Python 列表的第一个元素：

use pyo3::prelude::*;
use pyo3::types::PyList;


fn get_first_item<'py>(list: &Bound<'py, PyList>) -> PyResult<Bound<'py, PyAny>> {
    list.get_item(0)
}
Python::attach(|py| {
    let l = PyList::new(py, ["hello world"]).unwrap();
    assert!(get_first_item(&l).unwrap().eq("hello world").unwrap());
})

在 Python 对象类型之间转换

要将 Bound<'py, T> 智能指针转换为其他类型，请使用 .cast() 系列方法。这会将 &Bound<'py, T> 转换为另一个 &Bound<'py, U>，不转移所有权。另外有 .cast_into() 可将 Bound<'py, T> 转换为 Bound<'py, U> 并转移所有权。这些方法适用于所有实现了 PyTypeCheck trait 的类型 T。

转换为 Bound<'py, PyAny> 可使用 .as_any() 或 .into_any()。

例如，以下代码段展示了如何将 Bound<'py, PyAny> 转换为 Bound<'py, PyTuple>：

use pyo3::prelude::*;
use pyo3::types::PyTuple;
fn example<'py>(py: Python<'py>) -> PyResult<()> {
// 创建一个新的 Python `tuple`，并使用 `.into_any()` 擦除类型
let obj: Bound<'py, PyAny> = PyTuple::empty(py).into_any();


// 使用 `.cast()` 转换为 `PyTuple`，不转移所有权
let _: &Bound<'py, PyTuple> = obj.cast()?;


// 使用 `.cast_into()` 转换为 `PyTuple`，并转移所有权
let _: Bound<'py, PyTuple> = obj.cast_into()?;
Ok(())
}
Python::attach(example).unwrap()

自定义的 #[pyclass] 类型也实现了 PyTypeCheck，因此 .cast() 同样适用于这些类型。以下代码段与上方类似，但转换为自定义类型 MyClass：

use pyo3::prelude::*;


#[pyclass]
struct MyClass {}


fn example<'py>(py: Python<'py>) -> PyResult<()> {
// 创建一个新的 `MyClass` 实例，并使用 `.into_any()` 擦除类型
let obj: Bound<'py, PyAny> = Bound::new(py, MyClass {})?.into_any();


// 使用 `.cast()` 转换为 `MyClass`，不转移所有权
let _: &Bound<'py, MyClass> = obj.cast()?;


// 使用 `.cast_into()` 转换为 `MyClass`，并转移所有权
let _: Bound<'py, MyClass> = obj.cast_into()?;
Ok(())
}
Python::attach(example).unwrap()

从 Python 对象中提取 Rust 数据

若要从 Python 对象中提取 Rust 数据，请使用 .extract() 而不是 .cast()。此方法适用于所有实现了 FromPyObject trait 的类型。

例如，以下代码段从 Python 元组中提取一个 Rust 整数元组：

use pyo3::prelude::*;
use pyo3::types::PyTuple;
fn example<'py>(py: Python<'py>) -> PyResult<()> {
// 创建一个新的 Python `tuple`，并使用 `.into_any()` 擦除类型
let obj: Bound<'py, PyAny> = PyTuple::new(py, [1, 2, 3])?.into_any();


// 从 Python `tuple` 提取为 Rust `(i32, i32, i32)` 元组
let (x, y, z) = obj.extract::<(i32, i32, i32)>()?;
assert_eq!((x, y, z), (1, 2, 3));
Ok(())
}
Python::attach(example).unwrap()

为避免数据拷贝，#[pyclass] 类型可以直接引用存储在 Python 对象中的 Rust 数据，而无需调用 .extract()。更多细节请参见指南中类部分的相关文档。
